文档状态：草案

## 1. 验证目标

在正式开发前，先验证影响产品成立的关键技术假设。

## 2. 验证问题

| 编号 | 问题 | 验证方法 | 通过标准 |
|---|---|---|---|
| V-001 | 本地 JSONL 是否稳定包含 token_count | 连续使用 Codex 并采样日志 | 能解析 input、cached input、output |
| V-002 | rate_limits 字段是否能映射 5H | 与 Codex usage dashboard / `/status` 对账 | 误差可解释，字段稳定 |
| V-003 | 1W 剩余额度是否可稳定获得 | 连续多日采样并对账 | 能读取或明确降级为未知 |
| V-004 | Plus / Business 切换后快照如何归属账号 | 多账号切换测试 | 能区分旧账号和新账号 |
| V-005 | 官方登录/登出流程是否可被安全调用 | 测试 `codex login/status/logout` 或客户端等价能力 | 不直接操作 token 即可完成切换流程 |
| V-006 | credits 估算是否与官方 rate card 对齐 | 用样本 token 计算并与 dashboard 比对 | 估算逻辑可解释 |

# BKG-001 产品背景

文档编号：BKG-001  
文档状态：草案  
最后更新：2026-05-03  
关联文档：REQ-001

## 1. 背景概述

Codex 重度用户在长时间使用 ChatGPT Business 或 ChatGPT Plus 账号时，会受到 5 小时窗口和可能存在的 1 周窗口限制影响。用户希望在额度耗尽前获得明确提示，并能手动切换到另一个仍有额度的已授权账号，减少工作中断。

## 2. 用户背景

目标用户是企业 IT 管理者和 Codex 重度使用者，关注点包括：

- 企业协同办公系统产品与架构。
- IT 基础设施、运维稳定性、数据合规、信息安全。
- 多区域、多云、多门店场景下的持续交付和问题处理效率。
- 使用多个 ChatGPT Business / Plus 账号时的额度可见性与账号治理。

## 3. 产品机会

当前 Codex 客户端可以显示或暴露部分额度与 token 使用信息，但对多账号用户来说，缺少一个常驻、统一、可提醒、可审计的本地工具。

产品机会是做成 macOS 菜单栏工具：

- 常驻显示当前账号 5H / 1W 剩余额度。
- 展示本地 token 和 credits 消耗。
- 低额度时提醒。
- 推荐可用账号。
- 用户确认后执行账号切换流程。
- 切换后刷新新账号额度。

## 4. 合规定位

产品定位是“多账号额度治理”，不是“绕过额度限制”。必须坚持：

- 使用用户主动授权账号。
- 不静默轮换账号。
- 不复制、篡改或共享 OAuth token。
- 不把个人账号额度池化给他人使用。
- 不采集聊天内容、代码内容和私有仓库内容，除非用户明确打开相关功能。
